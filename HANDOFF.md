## 1. Project Overview

### What Zumo Is

A lightweight transcription-to-knowledge service. Users interact via a Telegram bot — they either send an audio/video file or ask the bot to pull a recording from their Zoom account. The bot transcribes, processes, and stores the output in the user's private GitHub repo. Each user gets a simple, password-protected web page (hosted on Netlify or similar) where they can browse and download their transcriptions.

### Who Uses It

Ben (primary user) + a small number of clients Ben manually onboards. This is NOT a SaaS product — there's no self-serve signup, no billing system, no enterprise features. Ben sets up each user manually.

### Core Flow

```
User sends file/link to Telegram bot
    → Bot asks 2-3 quick questions (who's talking, what type of session, any special needs)
    → Bot downloads/receives audio
    → Extract audio from video (if needed)
    → Remove silent segments >30 seconds
    → Send to Hebrew AI API for transcription
    → Send transcription to Claude API for structured analysis
    → Push both raw transcript + structured analysis to user's GitHub repo
    → Update index
    → Delete all local temp files
    → Bot sends confirmation with summary + link to web page
```

## 4. Telegram Bot — Detailed Spec

### Framework

`python-telegram-bot` (async). Same as tiktok-pipeline.

### User Authorization

Only users with matching `telegram_user_id` in `users/` config. Unknown users: "⛔ Not authorized."

### Input Handling

**File upload:** User sends video/audio file → bot processes it.

**Zoom pull:** User says "get my last meeting" → bot queries Zoom API → shows match as inline button → user confirms → bot downloads and processes.

### Interactive Flow (CRITICAL — Use Telegram's Full Capabilities)

**Step 1: Session Type** — Inline keyboard:
```
[📋 Team Meeting]  [🎓 Training Session]
[🤝 Client Call]   [📞 Phone Call]
[💡 Coaching]      [📝 Other]
```

**Step 2: Speakers** — Text input or [🔄 Detect from audio]

**Step 3: Special focus** — Inline buttons:
`[⏭ Skip — full analysis]` `[📝 Just action items]` `[📋 Meeting notes only]`

**Step 4: Processing** — Progress updates via message edits

**Step 5: Done** — Summary + inline buttons linking to files/dashboard

### Language

Bot communicates in the user's language (Hebrew/English).

## 6. Transcription — Hebrew AI

**POST /api/transcribe** — Submit audio file
```
Headers: Authorization: Bearer {api_key}
Body: file (multipart), language ("he" or "en")
Response: { success: true, transcription_id: "..." }
```

**GET /api/transcribe?id={id}** — Poll for result
```
Statuses: IN_QUEUE → PENDING → IN_PROGRESS → COMPLETED
Response: { text: "...", duration: 120 }
```

Poll every 5 seconds, timeout 30 minutes.
